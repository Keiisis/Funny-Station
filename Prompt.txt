🛠️ Roadmap Technique de l'Ingénieur (Plan de Bataille)
Pour que ce projet soit une réussite totale, voici les 5 piliers que vous devez construire dans l'ordre :

1. Le "Shell" (L'Expérience PS5)
L'UI doit être irréprochable. Utilisez des shaders WebGL pour l'arrière-plan mouvant afin de ne pas consommer de CPU, laissant ainsi toute la puissance pour les jeux. Le "Focus Management" est crucial : l'utilisateur ne doit jamais avoir besoin d'une souris.

2. La Machine Virtuelle Web (Universal Runner)
C'est le cœur du système.

Pour le C++/Rust, vous demanderez aux développeurs d'uploader un .wasm.
Pour le Python, vous intégrerez Pyodide qui permet de faire tourner des scripts Python à la vitesse du C dans le navigateur.
Le système doit injecter automatiquement un "SDK Funny Station" dans chaque jeu pour qu'ils puissent communiquer avec la console (ex: window.funnyStation.save()).
3. Le Système de Fichiers Virtuel (VFS)
Un jeu a besoin de sauvegarder des données. Comme on est sur le web, vous devez créer une couche qui fait croire au jeu qu'il écrit sur un disque dur (/home/user/save.dat), alors qu'en réalité, vous stockez cela dans Supabase Storage.

4. Le Dependency Manager (L'intelligence d'installation)
C'est votre demande spécifique. Quand un utilisateur upload un jeu :

Analyse : Le script scanne les fichiers (ex: cherche un requirements.txt ou un package.json).
Mapping : Il crée une configuration qui dit à la console : "Pour ce jeu, charge le runtime Python + la lib mathématique".
Isolation : Le jeu tourne dans un Web Worker pour que si le jeu plante, la console reste allumée.
5. Le Real-Time Multijoueur
Grâce à Supabase Realtime, vous pouvez créer des salons. Les joueurs s'envoient leurs positions de manettes via des "broadcasts" ultra-rapides, permettant de jouer ensemble sans avoir besoin d'un serveur de jeu coûteux.

1. ARCHITECTURE DU KERNEL (NOYAU SYSTÈME) :
- Framework : Next.js 14 (App Router) + TypeScript (Strict Mode).
- Universal Runtime Manager : Couche d'abstraction WebAssembly (WASM) supportant nativement :
    * C++/Rust (via Emscripten/Wasm-Bindgen)
    * Python (via Pyodide)
    * Lua (via Fengari)
    * Java (via CheerpJ)
- Virtual Memory & Swap : Système de gestion de mémoire utilisant IndexedDB pour 'swapper' les assets lourds sans saturer la RAM du navigateur.
- Isolation : Chaque jeu tourne dans un Web Worker dédié avec une Content Security Policy (CSP) stricte.

2. UI/UX "PS5 ULTIMATE REPLICA" :
- Moteur Graphique : Tailwind CSS + Framer Motion pour les transitions de 'Focus'.
- Background : Shaders WebGL/WebGPU dynamiques (ondes PS5) réagissant aux inputs.
- Audio Engine : Web Audio API pour le son spatialisé (3D Audio) et les SFX système.
- Input System : Support WebHID pour les manettes DualSense (Haptique/Triggers) et Xbox, avec mapping universel.

3. PIPELINE D'INSTALLATION INTELLIGENT :
- Analyseur d'Upload : Analyse automatique des fichiers .zip (package.json, requirements.txt, etc.).
- Dependency Injector : Génération dynamique d'Import Maps pour charger les runtimes et bibliothèques (Three.js, Phaser, etc.) depuis des CDNs optimisés.
- Virtual File System (VFS) : Persistence des sauvegardes via une couche IndexedDB synchronisée avec Supabase Storage.

4. SOCIAL & MULTIJOUEUR :
- Matchmaking : Lobbies en temps réel via Supabase Realtime (Presence & Broadcast).
- Trophy System : API interne permettant aux jeux d'enregistrer des succès sur le profil utilisateur.
